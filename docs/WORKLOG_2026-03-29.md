## 2. Built the Pipecat voice bot (`bot/`)

Created a standalone Python service that handles Twilio inbound calls end-to-end.

### Architecture
```
Twilio call → POST /twiml → TwiML with <Connect><Stream>
           → WebSocket /ws → Pipecat pipeline:
             Twilio audio → Deepgram STT → OpenAI LLM → Cartesia TTS → Twilio audio
```

### Files created
| File | Purpose |
|---|---|
| `bot/server.py` | FastAPI server: `/twiml` webhook + `/ws` WebSocket handler |
| `bot/bot.py` | Pipecat pipeline with STT, LLM, TTS, and tool calling |
| `bot/requirements.txt` | Python dependencies |
| `bot/Dockerfile` | Container build |
| `bot/railway.toml` | Railway deploy config (rootDirectory: bot, DOCKERFILE builder) |
| `bot/.env.example` | All required env vars documented |

### Services used
- **STT:** Deepgram (`nova-2-phonecall` model)
- **LLM:** OpenAI `gpt-4o-mini` (migrated from Anthropic Claude)
- **TTS:** Cartesia
- **Telephony:** Twilio Media Streams (WebSocket)

### LLM tools wired
- `check_availability` — calls `GET /api/locations` on the backend
- `book_reservation` — calls `POST /api/reservations`, then sends Twilio SMS confirmation

## 5. Bug fixes during deployment

### Fix 1 — TwiML: remove `<Pause>` after `<Connect>`
Twilio ignores all verbs after `<Connect>`; the malformed TwiML was silently preventing the media stream from opening.

### Fix 2 — TwiML: add `track="inbound_track"` to `<Stream>`
Required for Twilio to send inbound caller audio over the WebSocket.

### Fix 3 — Pipecat: fix CartesiaTTSService init
`InputParams(emotion=[])` caused Cartesia API to reject requests. Removed the parameter.

### Fix 4 — Pipecat: fix greeting trigger
Replaced `get_context_frame()` with `LLMMessagesFrame([...])` queued in `on_client_connected` — correct way to kick off the first LLM turn after the pipeline is running.

### Fix 5 — TwilioFrameSerializer: extract `stream_sid` before pipeline
`TwilioFrameSerializer(stream_sid)` requires the stream_sid at init time. Fixed the WebSocket handler to read Twilio's `connected` + `start` messages first, extract `streamSid`, then start the pipeline.

### Fix 6 — TwilioFrameSerializer: pass `call_sid` + Twilio credentials
Bot was crashing with `auto_hang_up is enabled but missing required parameters`. Fixed by extracting `callSid` from the start message and passing `call_sid`, `account_sid`, `auth_token` to the serializer.
