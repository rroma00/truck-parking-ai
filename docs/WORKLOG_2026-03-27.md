## Root Causes Identified

### SPA route deployment issue

- Vercel was serving the frontend as a static app without an SPA rewrite, so direct requests to nested routes could 404.

### Onboarding production crash

- Public onboarding was mounting inside `LotProvider`, which triggered `/lots` on initial render.
- The backend still has legacy `Lots`-table dependencies in some paths, producing errors like:
  - `GET /lots 500`
  - `Could not find table public.Lots in schema cache`
  - `Lot not found`

### Onboarding import build failure

- The import path casing was already correct.
- The missing tracked file on GitHub caused Vercel to fail to resolve `./pages/Onboarding` in Linux production builds.
