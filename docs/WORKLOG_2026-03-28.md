## 🚀 Summary

Today focused on replacing Anthropic (Claude) with OpenAI as the main LLM provider for the phone agent.

---

## 🔄 LLM Provider Migration

### Before
- Anthropic (Claude) used as AI brain

### After
- OpenAI Responses API
- Model: `gpt-5.4-mini`

---

## 🧠 Reason for Change

- Reduce cost (Claude usage hitting limits / billing)
- Better control over structured workflows
- More predictable tool usage
- Easier scaling for multiple parking lots (20–30+)

---

## ⚙️ Implementation Details

- Replaced Anthropic SDK with OpenAI SDK (`openai`)
- Introduced `OPENAI_API_KEY` environment variable
- Switched model calls to:
  - `client.responses.create(...)`
- Standardized prompt format:
  - system + user structure
- Ensured compatibility with existing Pipecat flow

---

## 🧩 Architecture Decision

Chose **text-based pipeline instead of realtime voice model**

Pipeline:

Phone/SMS → STT → OpenAI (text) → Tools → TTS → Response

---

## 💰 Cost Consideration

- `gpt-5.4-mini` selected as default
- ~ $0.002 per call
- Scales efficiently for 20–30 parking lots
